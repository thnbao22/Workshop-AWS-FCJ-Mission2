Terraform

Tổng quan

Với thư mục terraform, chúng ta có thể gọi tới một số mô-đun như Mạng, Điện toán và Cân bằng tải. Chúng ta không cần phải điều hướng đến từng thư mục và chạy terraform plan, terraform apply để tạo từng tài nguyên.

Cài đặt các tập tin

Trong thư mục terraform, tạo ba tập tin và đặt tên tương ứng là main.tf, outputs.tf và provider.tf.

Thiết lập nhà cung cấp

Trong Terraform, nhà cung cấp là một plugin hoạt động như một giao diện giữa Terraform và một dịch vụ hoặc nền tảng bên ngoài. Các nhà cung cấp cho phép Terraform quản lý tài nguyên, tương tác với API và thực hiện các hoạt động trên nhiều nền tảng đám mây, dịch vụ cơ sở hạ tầng và hệ thống của bên thứ ba.

Một số nhà cung cấp Terraform phổ biến:

  • azurerm: Azure
  • aws: Amazon Web Services
  • google: Google Cloud Platform

Ở đây ta triển khai trên AWS nên cần cấu hình trong file provider.tf. Chúng ta sẽ tạo ra tài nguyên ở khu vực Singapore.

terraform {
  required_providers {
    aws = {
      source = "hashicorp/aws"
      version = "5.43.0"
    } 
  }
}
provider "aws" {
  region = "ap-southeast-1"
}

Thiếp lập các mô-đun

Như ta đã nói trước đây, bạn không cần phải đi đến từng mô-đun và chạy terraform plan, terraform apply để tạo từng tài nguyên. Bạn có thể xác định các mô-đun trong tệp main.tf.

# Define our modules
module "Networking" {
  source = "../modules/Networking"
  cidr_block = "10.10.0.0/16"
}

module "Compute" {
  source              = "../modules/Compute"
  instance_type       = "t2.micro"
  image_id            = "ami-04f73ca9a4310089f"
  keypair_name        = "workshop-keypair-2"
  web_server_sg_id    = module.Networking.public_sg_id
  public_subnet_1_id  = module.Networking.public_subnet_1_id
  public_subnet_2_id  = module.Networking.public_subnet_2_id
  alb_tg_arn          = module.LoadBalancing.alb_tg_arn
}

module "LoadBalancing" {
  source                = "../modules/LoadBalancing"
  one_tier_vpc_id       = module.Networking.vpc_id
  public_subnet_1_id    = module.Networking.public_subnet_1_id
  public_subnet_2_id    = module.Networking.public_subnet_2_id
  alb_security_group_id = module.Networking.alb_sg_id
  port                  = 80
  protocol              = "HTTP"
}

Mô-đun mạng

Trong mô-đun Mạng, chúng ta chỉ cần nhập CIDR của VPC là 10.10.0.0/16.

Mô-đun điện toán

Ở đây, chúng tôi sử dụng ID hình ảnh của Amazon Linux 2 AMI và loại phiên bản là t2.micro. Bạn có thể lấy những thông tin này từ AMI.

Về keypair_name, hãy truy cập AWS Management Console và tìm kiếm EC2 (Hãy nhớ thay đổi khu vực của bạn thành Singapore) sau đó nhấp vào Key Pairs.

  • Nhấn vào Create key pair

Cấu hình như bên dưới.

Sau khi tạo key pair thành công, bạn sẽ thấy một thông báo.

Về web_server_sg_id, public_subnet_1_id, public_subnet_2_id và lb_target_group_name, chúng ta sẽ lấy đầu vào từ module Networking.

Mô đun Cân bằng tải

Chúng ta chỉ xác định giao thức HTTTP trên cổng 80. Về one_tier_vpc_id, public_subnet_1_id, public_subnet_2_id và alb_security_group_id, chúng ta cũng sẽ lấy thông tin đầu vào từ mô-đun Mạng

Kết luận

Bằng cách sử dụng lợi thế này, bạn có thể tự động tạo tài nguyên bằng cách sử dụng Terraform mà không cần tạo chúng theo cách thủ công trên AWS Management Console.

Terraform sẽ xác định thứ tự các tài nguyên cần được tạo. Ban đầu, các tài nguyên trong phần Mạng sẽ được tạo trước tiên, tiếp theo là các tài nguyên trong phần Tính toán và cuối cùng là tài nguyên Cân bằng tải sẽ được thiết lập.

Sau đó, chúng ta có thể kiểm tra bằng cách sử dụng DNS của ALB để kiểm tra máy chủ web của mình. Chúng ta cần xuất DNS của ALB trong tệp outputs.tf.

output "alb_dns" {
  value = module.LoadBalancing.alb_dns
}
